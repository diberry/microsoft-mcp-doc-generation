## Acrolinx Compliance Guidelines

Apply these rules to all generated text:

- **Present tense:** Use present tense. Write "returns" not "will return", "creates" not "will create".
- **Contractions:** Use contractions. Write "doesn't" not "does not", "isn't" not "is not", "don't" not "do not".
- **Active voice:** Use active voice. Write "The tool lists resources" not "Resources are listed by the tool".
- **Introductory commas:** Add commas after introductory phrases: "For example, list the resources", "By default, the tool returns all items".
- **No first person:** Never use "we", "our", or "us". Use "the tool", "this tool", or "you".
- **Acronym expansion:** Define acronyms on first use: "role-based access control (RBAC)" then "RBAC" afterward.
- **Relative URLs:** Use site-root-relative URLs: "/azure/..." not "https://learn.microsoft.com/azure/...".
- **Sentence length:** Keep sentences under 35 words. Split long sentences into two.
- **No wordy phrases:** Use "to" not "in order to", "use" not "utilize", "before" not "prior to".
- **Brand compliance:** Use official Azure service names: "Azure Cosmos DB" not "CosmosDB", "Azure Kubernetes Service" not "AKS" on first use.
