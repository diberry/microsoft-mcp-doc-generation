## SEO Differentiation Rules (CRITICAL)

These articles document Azure MCP Server tools, NOT the Azure services themselves. The content must complement official Azure service docs, never compete with them.

- **Title and H1**: MUST lead with "Azure MCP Server" — never lead with the Azure service name (e.g., never "Azure AI Search tools for..." — always "Azure MCP Server tools for Azure AI Search...")
- **Description**: MUST lead with "Use Azure MCP Server tools" or similar MCP-first phrasing

## Requirements

1. **Frontmatter**:
   - `title`: Must follow pattern "Azure MCP Server tools for {ServiceName}"
   - `description`: Must lead with "Use Azure MCP Server tools to..." — MCP Server product name must appear before the service name
   - `ms.service`: Always "azure-mcp-server"
   - `ms.topic`: Always "concept-article"
   - `tool_count`: Exact number provided in user prompt
   - `mcp-cli.version`: CLI version string provided in user prompt

2. **H1 Heading**:
   - Must match the title exactly
   - Format: "Azure MCP Server tools for {ServiceName}"

3. **Introduction Paragraph 1**:
   - Start with "The Azure MCP Server lets you manage..."
   - Use the **Available Operations** verb list provided in the user prompt to describe what operations are available. Include all provided verbs (e.g., "create, get, list, update, and delete") rather than guessing capabilities. Do not omit any verbs from the provided list.
   - End with "with natural language prompts"

4. **Introduction Paragraph 2**:
   - One sentence describing the service
   - Include a link to official Microsoft documentation: `[{ServiceName} documentation]({url})`
   - Use actual Microsoft Learn URLs (e.g., /azure/storage/, /azure/cosmos-db/)

5. **Include Statement**:
   - Always include: `[!INCLUDE [tip-about-params](../includes/tools/parameter-consideration.md)]`
   - No extra text after this line

## Style Guidelines

- Use sentence case for titles and headings
- Keep descriptions concise (1-2 sentences)
- Use active voice ("lets you manage" not "allows management of")
- Be specific about capabilities (don't just say "operations")
- Follow Microsoft Learn writing style (professional, clear, user-focused)

## Output Only Markdown

- Do NOT include explanations or meta-commentary
- Do NOT wrap in code fences
- Output ONLY the markdown content specified above
