## Important Note

If this tool has **0 required parameters** (after excluding infrastructure parameters listed in Rule 1), then:
- All example prompts are automatically valid
- Return the JSON with empty validation array and isValid=true
- No further analysis is needed

Example: If the tool only has optional and infrastructure parameters, return:
```json
{
  "toolName": "example-tool",
  "requiredParameters": [],
  "totalPrompts": 5,
  "validPrompts": 5,
  "invalidPrompts": 0,
  "isValid": true,
  "validation": [],
  "summary": "No required parameters to validate - all prompts are valid",
  "recommendations": []
}
```

## Validation Instructions

Validate the example prompts in this tool file according to the rules in the system prompt.

**Key reminders:**
- Exclude infrastructure parameters (subscription, tenant, auth-method, retry-*, output, verbose, debug, help) from validation
- Recognize natural language parameter variations (e.g., "resource group 'rg-prod'" for `resource-group`)
- Parameter values should be in single quotes with concrete values

## Output Format

Return your validation results as a JSON object with the structure shown in the system prompt, including:
- toolName, requiredParameters, totalPrompts, validPrompts, invalidPrompts, isValid
- validation array with details for each prompt
- summary and recommendations

Be thorough and ensure all required parameters are accounted for in each prompt.
